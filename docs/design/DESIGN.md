# Rizhi 设计规范

> 注意：本文档为早期 V1 设计规范。后续 UI 重构以 [UI_BASELINE_V2.md](./UI_BASELINE_V2.md) 为准；本文仅作为历史参考。

## 1. 设计基调

Rizhi 的界面应保持“轻量、清晰、可信赖”的 PC 管理后台气质。

参考效果图的整体风格：

- 浅色背景
- 左侧固定导航
- 顶部工具栏
- 白色卡片承载信息
- 低饱和蓝色作为主色
- 红色表示支出、负债、亏损和风险
- 绿色表示收入、正向增长、正常状态和保修剩余
- 页面留白充足，信息密度适中

关键词：

```text
清爽、理性、克制、数据感、生活资产管理、轻后台
```

## 2. 页面布局

### 2.1 整体结构

PC 端采用固定侧边栏 + 顶部栏 + 内容区布局。

```text
┌──────────────┬────────────────────────────────────┐
│ 侧边导航      │ 顶部栏                              │
│              ├────────────────────────────────────┤
│              │ 页面内容区                           │
└──────────────┴────────────────────────────────────┘
```

推荐尺寸：

- 侧边栏宽度：112px
- 顶部栏高度：56px
- 内容区左右内边距：24px
- 内容区上下内边距：20px
- 页面最小宽度：1280px
- 主要内容最大宽度：不强制限制，跟随浏览器宽度

### 2.2 侧边栏

侧边栏用于承载一级导航。

样式：

- 背景色：`#FFFFFF`
- 右边框：`1px solid #E8EDF5`
- Logo 区高度：56px
- 导航项高度：40px
- 导航项左右内边距：12px
- 导航项圆角：6px
- 导航项间距：4px

导航项状态：

- 默认：透明背景，文字 `#4B5565`
- 悬停：背景 `#F5F8FF`
- 选中：背景 `#EAF2FF`，文字 `#1677FF`
- 图标默认：`#6B7280`
- 图标选中：`#1677FF`

侧边栏底部放置：

- 设置
- 帮助与反馈

### 2.3 顶部栏

顶部栏承载全局搜索、通知和用户信息。

样式：

- 高度：56px
- 背景色：`#FFFFFF`
- 下边框：`1px solid #E8EDF5`
- 内容左右内边距：24px

顶部栏元素：

- 全局搜索框
- 通知图标
- 消息图标
- 用户头像
- 用户名称
- 下拉箭头

## 3. 颜色规范

### 3.1 主色

主色用于主要按钮、选中状态、链接和重点操作。

```text
Primary 600: #1677FF
Primary 500: #2F80FF
Primary 100: #EAF2FF
Primary 50:  #F5F8FF
```

### 3.2 中性色

```text
Text Primary:   #111827
Text Secondary: #4B5565
Text Tertiary:  #8A94A6
Text Disabled:  #B8C0CC

Border:         #E8EDF5
Border Strong:  #D6DDE8
Page Background:#F5F7FB
Card Background:#FFFFFF
Hover Background:#F8FAFD
```

### 3.3 语义色

收入、正向增长、可用状态：

```text
Success:        #16A36A
Success Light:  #EAFBF2
```

支出、负债、亏损、风险：

```text
Danger:         #F04438
Danger Light:   #FFF1F0
```

提醒、即将过期、需要关注：

```text
Warning:        #F59E0B
Warning Light:  #FFF7E6
```

普通信息：

```text
Info:           #1677FF
Info Light:     #EAF2FF
```

### 3.4 金额颜色规则

- 收入金额：绿色 `#16A36A`
- 支出金额：红色 `#F04438`
- 负债金额：红色 `#F04438`
- 普通资产金额：深色 `#111827`
- 增长百分比：绿色
- 下降百分比：红色
- 中性数据：灰色或深色

## 4. 字体规范

推荐字体：

```css
font-family: Inter, "PingFang SC", "Microsoft YaHei", Arial, sans-serif;
```

字号层级：

```text
页面标题：20px / 28px，600
区块标题：16px / 24px，600
卡片标题：14px / 22px，600
正文：14px / 22px，400
辅助文字：12px / 18px，400
表格文字：13px / 20px，400
金额大字：20px / 28px，600
金额中号：16px / 24px，600
标签文字：12px / 18px，500
```

数字建议使用等宽数字特性：

```css
font-variant-numeric: tabular-nums;
```

## 5. 间距规范

基础间距单位使用 4px。

常用间距：

```text
4px：极小元素间距
8px：表单项内部、标签间距
12px：卡片内部小间距
16px：卡片内边距、组件间距
20px：区块间距
24px：页面内容边距
32px：大区块间距
```

卡片内边距：

- 小卡片：16px
- 普通卡片：20px
- 复杂卡片：24px

## 6. 圆角与阴影

### 6.1 圆角

整体圆角保持克制。

```text
按钮：6px
输入框：6px
标签：999px 或 4px
卡片：8px
弹窗：10px
图片：6px
```

不要使用过大的圆角，避免变成移动端卡通风格。

### 6.2 阴影

默认以边框为主，阴影只用于悬浮层、下拉菜单和重点卡片。

```text
Card Shadow:    0 4px 16px rgba(17, 24, 39, 0.04)
Popover Shadow: 0 8px 24px rgba(17, 24, 39, 0.10)
Modal Shadow:   0 16px 48px rgba(17, 24, 39, 0.16)
```

## 7. 按钮规范

### 7.1 按钮类型

主要按钮 Primary：

- 用于页面主操作，比如“新增资产”“新增记账”“添加账户”
- 背景：`#1677FF`
- 文字：`#FFFFFF`
- 悬停：`#2F80FF`
- 禁用：背景 `#B8D4FF`，文字 `#FFFFFF`

次要按钮 Secondary：

- 用于次级操作，比如“导出”“筛选”“重置”
- 背景：`#FFFFFF`
- 边框：`#D6DDE8`
- 文字：`#344054`
- 悬停背景：`#F8FAFD`

文本按钮 Text：

- 用于轻量操作，比如“查看全部”“编辑”
- 背景：透明
- 文字：`#1677FF`
- 悬停背景：`#F5F8FF`

危险按钮 Danger：

- 用于删除、清空、取消关联等危险操作
- 背景：`#F04438`
- 文字：`#FFFFFF`
- 悬停：`#D92D20`

### 7.2 按钮尺寸

```text
小按钮：高度 28px，左右 padding 10px，字号 12px
默认按钮：高度 32px，左右 padding 12px，字号 13px
大按钮：高度 36px，左右 padding 16px，字号 14px
```

按钮图标：

- 图标尺寸：16px
- 图标与文字间距：6px

页面右上角主按钮使用默认按钮或大按钮。

## 8. 输入与筛选控件

### 8.1 输入框

样式：

- 高度：32px
- 背景：`#FFFFFF`
- 边框：`1px solid #D6DDE8`
- 圆角：6px
- 左右内边距：12px
- 文字：`#111827`
- 占位文字：`#B8C0CC`
- 聚焦边框：`#1677FF`
- 聚焦阴影：`0 0 0 3px rgba(22, 119, 255, 0.10)`

### 8.2 搜索框

全局搜索框：

- 宽度：360px
- 高度：32px
- 背景：`#F5F7FB`
- 边框：透明或 `#E8EDF5`
- 图标尺寸：14px

列表页搜索框：

- 宽度：240px 至 320px
- 高度：32px

### 8.3 下拉筛选

- 高度：32px
- 圆角：6px
- 边框：`#D6DDE8`
- 默认背景：`#FFFFFF`
- 菜单阴影：使用 Popover Shadow

筛选区域建议横向排列，放在列表上方。

## 9. 卡片规范

卡片是 Rizhi 的核心信息容器。

默认卡片：

- 背景：`#FFFFFF`
- 边框：`1px solid #E8EDF5`
- 圆角：8px
- 阴影：`0 4px 16px rgba(17, 24, 39, 0.04)`
- 内边距：16px 或 20px

卡片标题：

- 字号：14px
- 字重：600
- 颜色：`#111827`

卡片辅助说明：

- 字号：12px
- 颜色：`#8A94A6`

### 9.1 数据概览卡片

用于展示净资产、总资产、负债、本月收入、本月支出等。

结构：

- 标题
- 主金额
- 环比或说明
- 可选图标或进度环

样式：

- 卡片高度：88px 至 104px
- 主金额字号：20px
- 标题字号：13px
- 变化说明字号：12px

### 9.2 资产卡片

用于资产列表页的网格卡片。

结构：

- 资产图片
- 资产名称
- 品牌或分类
- 当前价格或总成本
- 持有天数
- 日均成本
- 状态标签
- 更多操作按钮

样式：

- 卡片宽度：自适应网格
- 图片比例：4:3 或固定 120px x 90px
- 图片圆角：6px
- 卡片内边距：14px
- 卡片间距：16px

悬停状态：

- 边框变为 `#BBD5FF`
- 阴影略增强

## 10. 表格规范

表格用于记账列表、附加项列表、资金流水等高密度信息。

样式：

- 表头背景：`#F8FAFD`
- 表头文字：12px，500，`#667085`
- 表格正文：13px，`#344054`
- 行高：44px
- 单元格左右 padding：12px
- 行分割线：`1px solid #E8EDF5`
- 悬停行背景：`#F8FAFD`

金额列：

- 右对齐
- 使用等宽数字
- 收入绿色
- 支出红色
- 普通金额深色

操作列：

- 放在最右侧
- 图标按钮尺寸 28px
- 默认灰色，悬停蓝色

空状态：

- 居中展示
- 说明文字 13px，`#8A94A6`
- 可带一个主操作按钮

## 11. 标签与状态

### 11.1 普通标签

用于分类、状态、资产类型。

样式：

- 高度：22px
- 左右 padding：8px
- 圆角：4px 或 999px
- 字号：12px
- 字重：500

### 11.2 状态标签

使用中：

- 背景：`#EAFBF2`
- 文字：`#16A36A`

闲置：

- 背景：`#F5F7FB`
- 文字：`#667085`

即将过保：

- 背景：`#FFF7E6`
- 文字：`#F59E0B`

已过保 / 支出 / 负债：

- 背景：`#FFF1F0`
- 文字：`#F04438`

进行中 / 订阅中：

- 背景：`#F1EFFF`
- 文字：`#635BFF`

## 12. 图标规范

建议使用 `lucide-react` 或同类线性图标库。

图标风格：

- 线性
- 统一 1.75px 至 2px stroke
- 不使用大面积填充图标

尺寸：

- 导航图标：16px
- 表格操作图标：14px
- 按钮图标：16px
- 概览卡片图标：18px

颜色：

- 默认：`#6B7280`
- 选中：`#1677FF`
- 危险：`#F04438`
- 成功：`#16A36A`

## 13. 图片规范

资产图片应清晰展示物品本身，不使用抽象占位图作为正式内容。

资产图片：

- 背景：浅灰或白色
- 圆角：6px
- 填充方式：`object-fit: cover`
- 缺省状态可使用简单物品图标和浅灰背景

详情页主图：

- 建议尺寸：240px x 240px
- 背景：`#F5F7FB`
- 圆角：8px

缩略图：

- 尺寸：40px x 40px 或 48px x 48px
- 圆角：4px

## 14. 页面级规范

### 14.1 看板页

布局：

- 顶部为数据总览卡片横排
- 中部为三列卡片区
- 底部为图表、排行、最近记录

推荐：

- 第一行展示 5 至 6 个概览卡片
- 信息卡片不要嵌套卡片
- 图表保持轻量，线条细，颜色控制在蓝、红、绿三类

### 14.2 资产列表页

布局：

- 顶部为分类 Tab 和新增按钮
- 第二行为筛选器和视图切换
- 主体为资产卡片网格或表格
- 底部为分页

资产卡片网格：

- 每行 4 至 5 个
- 间距 16px
- 卡片高度保持一致

### 14.3 资产详情页

布局：

- 顶部为面包屑和操作按钮
- 左侧为资产图片
- 中间为资产基础信息
- 右侧为成本摘要卡片
- 下方使用 Tab 展示附加项、关联账单、使用记录、历史记录

成本摘要应突出：

- 原始购买价
- 附加项合计
- 转卖收入
- 总成本
- 日均成本

### 14.4 记账页

布局：

- 顶部为交易记录 / 日历视图切换
- 筛选区位于表格上方
- 右上角放“新增记账”
- 主体使用表格

记账表格应突出：

- 日期
- 类型
- 分类
- 商家或备注
- 账户
- 关联资产
- 金额
- 操作

### 14.5 资金页

布局：

- 顶部为资金总览卡片
- 中部左右分区：资产账户、负债账户
- 底部展示还款提醒和最近资金变动

正向资产与负债应有明确视觉区分：

- 资产区使用蓝色和绿色趋势
- 负债区使用红色趋势和浅红背景

## 15. 交互状态

所有可点击元素都必须有明确状态。

按钮状态：

- 默认
- 悬停
- 按下
- 禁用
- 加载中

输入状态：

- 默认
- 聚焦
- 错误
- 禁用

卡片状态：

- 默认
- 悬停
- 选中

表格行状态：

- 默认
- 悬停
- 选中

错误提示：

- 表单项错误显示在输入框下方
- 字号 12px
- 颜色 `#F04438`

## 16. 弹窗与抽屉

新增、编辑类操作优先使用抽屉或弹窗。

推荐规则：

- 简单表单：弹窗
- 复杂表单：右侧抽屉
- 资产新增、记账新增、账户新增：可使用右侧抽屉
- 删除确认：弹窗

弹窗：

- 宽度：420px 至 560px
- 圆角：10px
- 内边距：24px

右侧抽屉：

- 宽度：480px 至 640px
- 背景：`#FFFFFF`
- 左边框：`1px solid #E8EDF5`

底部操作区：

- 右对齐
- 主按钮在右
- 次按钮在左

## 17. 空状态与加载状态

空状态文案应短，不要解释过多。

示例：

- 暂无物品资产
- 暂无记账记录
- 暂无资金账户
- 还没有附加项

空状态操作：

- 资产页：新增资产
- 记账页：新增记账
- 资金页：添加账户

加载状态：

- 页面初始加载使用骨架屏
- 表格加载使用行骨架
- 按钮提交使用 loading 状态

## 19. 响应式规则

第一版以 PC 端为主。

推荐断点：

```text
1280px：基础 PC 布局
1440px：标准桌面布局
1920px：宽屏布局
```

当宽度不足时：

- 侧边栏保持固定宽度
- 卡片网格减少列数
- 表格允许横向滚动
- 顶部搜索框可缩短

暂不要求完整移动端体验，但页面不能出现明显重叠、遮挡或内容溢出。


## 22. 设计禁忌

- 不使用大面积渐变背景。
- 不使用过重阴影。
- 不使用过大的圆角。
- 不把每个区域都做成厚重卡片。
- 不使用过多颜色表达数据。
- 不用红色表达普通提醒，红色只用于支出、负债、错误、风险。
- 不让主按钮在同一页面出现过多。
- 不在表格里塞入过长说明文字。
