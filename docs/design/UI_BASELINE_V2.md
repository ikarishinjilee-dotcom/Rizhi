# 日值 V2 全局 UI 基准

本文档用于后续 UI 重构。所有 PC 页面、弹窗、基础组件和业务组件，优先按本基准统一。

## 1. 设计定位

V2 UI 的方向是：个人资产与记账的轻量数据工作台。

关键词：

- 干净
- 轻盈
- 数据感
- 低饱和
- 柔和卡片
- 高信息密度但不压迫

整体避免传统后台的厚重边框和密集表格感，也避免移动 App 式的大面积装饰。界面应更像一个可长期使用的个人财务与资产工作台。

## 2. 页面结构

PC 页面统一采用以下信息节奏：

1. 页面标题区
2. 顶部操作区 / 搜索区
3. 核心统计区
4. 分析图表区
5. 筛选区
6. 业务列表区

页面内容宽度不应过窄。主内容区域左右保留 28px 到 32px 间距，模块间距以 24px 为主。

## 3. 颜色系统

基础色：

| Token | Value | 用途 |
| --- | --- | --- |
| `--color-bg-page` | `#F5F7FB` | 页面背景 |
| `--color-bg-card` | `#FFFFFF` | 卡片背景 |
| `--color-border` | `#E2E8F0` | 默认边框 |
| `--color-border-soft` | `#E8EEF7` | 轻边框 |
| `--color-text-primary` | `#0F172A` | 一级文字 |
| `--color-text-secondary` | `#475569` | 二级文字 |
| `--color-text-muted` | `#94A3B8` | 辅助文字 |

品牌与状态色：

| Token | Value | 用途 |
| --- | --- | --- |
| `--color-primary` | `#2674FF` | 主按钮、选中态、重点链接 |
| `--color-primary-start` | `#1F6BFF` | 主蓝渐变起点 |
| `--color-primary-end` | `#6EA8FF` | 主蓝渐变终点 |
| `--color-success` | `#16B878` | 使用中、收入、正向变化 |
| `--color-warning` | `#F59E0B` | 闲置、待处理、临期 |
| `--color-danger` | `#EF4444` | 删除、亏损、过保、风险 |
| `--color-purple` | `#7C3AED` | 特殊提醒、信用额度 |

颜色使用规则：

- 蓝色只用于主要操作、选中态、重点资产数据。
- 绿色用于正向状态和正向金额。
- 橙色用于临期、待处理、闲置。
- 红色只用于危险、损失、删除和负向风险。
- 大面积背景优先使用浅色渐变，不使用高饱和纯色铺满。

## 4. 字体层级

| 场景 | 字号 | 行高 | 字重 |
| --- | --- | --- | --- |
| 页面标题 | 28px | 34px | 700 |
| 页面副标题 | 15px | 22px | 400 |
| 卡片标题 | 16px | 24px | 700 |
| 数据大数字 | 30px | 38px | 700 |
| 普通正文 | 14px | 22px | 400 |
| 辅助说明 | 13px | 20px | 400 |
| 表格/列表小字 | 12px | 18px | 400 |

原则：

- 金额、数量、核心指标可以放大。
- 说明文字要轻，不和核心数据抢视觉焦点。
- 页面内不要出现过多同级粗体。

## 5. 间距系统

使用 4px 基础倍数：

```text
4 / 8 / 12 / 16 / 20 / 24 / 32 / 40
```

推荐值：

| 场景 | 间距 |
| --- | --- |
| 页面左右 padding | 28px - 32px |
| 页面模块间距 | 24px |
| 卡片内边距 | 20px - 24px |
| 卡片网格 gap | 16px |
| 表单字段 gap | 16px |
| 表格行高 | 48px - 56px |

## 6. 圆角

| 场景 | 圆角 |
| --- | --- |
| 页面主卡片 | 16px |
| 数据卡片 | 16px |
| 业务卡片 | 16px |
| 按钮 | 12px |
| 输入框 / 选择框 | 12px |
| 弹窗 | 20px |
| 标签 | 999px |

整体使用柔和圆角，但避免过度拟物。

## 7. 阴影与边框

推荐 token：

```css
--shadow-card: 0 8px 24px rgba(15, 23, 42, 0.06);
--shadow-float: 0 16px 40px rgba(15, 23, 42, 0.12);
```

卡片默认样式：

```css
background: #ffffff;
border: 1px solid #e8eef7;
box-shadow: 0 8px 24px rgba(15, 23, 42, 0.05);
border-radius: 16px;
```

悬浮态：

- `transform: translateY(-2px)`
- 阴影略微增强
- 不改变布局尺寸

## 8. 按钮

主按钮：

- 高度：40px 到 44px
- 圆角：12px
- 背景：主蓝或主蓝渐变
- 字重：600
- 可包含左侧图标

次按钮：

- 白底
- 浅边框
- hover 使用浅蓝背景

危险按钮：

- 红色文字或红底
- 只用于删除、清空、不可逆操作

## 9. 输入框与搜索框

搜索框：

- 高度：40px
- 圆角：999px 或 14px
- 左侧搜索图标
- 白底、浅边框
- focus 使用蓝色描边和极浅蓝阴影

普通输入框：

- 高度：40px
- 圆角：12px
- 边框：`#DDE5F0`
- focus：`#2674FF`

表单错误：

- 红色错误文字显示在字段下方
- 错误状态不能撑乱相邻字段高度，双列表单应保留统一行高

## 10. 卡片类型

### 数据卡片

用于顶部统计。

- 高度：120px 到 140px
- 支持浅色渐变背景
- 可以包含图标、小折线、大数字、同比说明
- 关键数据优先，说明文字次级

### 分析卡片

用于图表。

- 白底
- 轻边框
- 标题 + 筛选器 + 图表
- 高度：220px 到 260px

### 业务卡片

用于资产、账户、记录等业务对象。

- 图片或图标区域明确
- 左上角显示状态标签
- 右上角放更多菜单
- 底部显示核心金额和关键指标

## 11. 标签

状态标签统一为轻量胶囊：

| 状态 | 样式 |
| --- | --- |
| 使用中 | 绿字 + 浅绿底 |
| 即将过保 | 橙字 + 浅橙底 |
| 已过保 | 红字 + 浅红底 |
| 闲置 | 灰蓝字 + 浅灰底 |
| 收入 | 绿字 + 浅绿底 |
| 支出 | 红字 + 浅红底 |

不要使用强实心色块作为普通状态标签。

## 12. 图表

图表风格保持轻量：

- 折线图使用主蓝。
- 面积图使用浅蓝渐变。
- 环形图使用低饱和多色。
- 图例右侧或下方排列，数字对齐。
- 图表卡片内部必须有足够留白。

图表是辅助信息，不应压过业务列表。

## 13. 弹窗

弹窗统一：

- 居中显示
- 圆角：20px
- 轻阴影
- 内容宽度按任务复杂度分级：
  - 小确认：420px
  - 普通编辑：640px
  - 复杂业务：860px 到 960px
- 表单优先使用 2 列布局
- 弹窗底部固定操作区

重要业务弹窗可以使用轻渐变头部，但不要所有弹窗都大面积渐变。

## 14. 页面适配策略

优先按 1440px 到 1920px PC 宽度设计。

响应式规则：

- 大于 1280px：多列卡片和图表。
- 1024px 到 1280px：减少列数，保持内容不挤压。
- 小于 1024px：侧栏可收起，卡片改为单列或双列。

## 15. 后续重构顺序

建议执行顺序：

1. 重构全局 tokens。
2. 重构基础组件：`RButton`、`RInput`、`RSelect`、`RCard`、`RModal`、`RTag`。
3. 重构应用框架：侧栏、顶部栏、页面容器。
4. 重做物品资产页。
5. 重做资金页。
6. 重做记账页。
7. 重做资产详情页。
8. 重做看板页。
9. 重做设置中心和管理中心。

每次只改一个页面或一组基础组件，避免 UI 重构和业务逻辑重构混在一起。
