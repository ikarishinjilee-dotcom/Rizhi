# 日值开发路线图

> 路线图以当前 Web + uniCloud 版本为基线，不再把 Fastify/SQLite 迁移作为未来阶段。

## 1. 已完成

- PC Web Vue 3 应用和统一路由。
- IndexedDB/Dexie 本地数据源与 uniCloud HTTP 数据源切换。
- uni-id 注册、登录、身份刷新和用户隔离。
- 资产、附加项、记账、资金账户、转账、还款和分类联动。
- 备份、恢复、附件上传和历史数据迁移入口。
- 管理员用户管理、业务字典和站点品牌配置。
- 测试线和正式线 Web Hosting 部署脚本。

## 2. 当前优先级

### P0：上线稳定性

- 建立正式环境健康检查、登录、数据初始化和关键写入的发布后检查表。
- 完善云函数错误日志、请求耗时和图片上传失败提示。
- 明确静态 Web 产物与 uni-app H5 产物的发布边界。

### P1：数据可靠性

- 增加云端 API 的集成测试和用户隔离测试。
- 完善数据库 Schema 索引、数据迁移和异常恢复文档。
- 增加导入前预览、冲突提示和备份版本校验。

### P1：体验完善

- 优化大页面首屏和大 chunk，按路由进一步拆分加载。
- 完善移动端 Web 响应式适配和空状态。
- 增加品牌配置的预览、回滚和默认值恢复。

### P2：跨端收敛

- 让 uni-app 客户端逐步复用稳定的领域模型和 API 契约。
- 评估将跨端共享模型提取到 `packages/` 的时机。
- 补齐 Android、iOS 和小程序端的发布检查。

## 3. 暂不安排

- 恢复 Fastify + SQLite 正式运行时。
- 重新实现 SQLite 结构化迁移路线。
- 在没有明确业务需求前引入复杂协作、订阅和支付体系。
