# 日值开发路线图

这份路线图用于回答“下一步做什么”。项目按阶段推进，每个阶段都要有明确交付物。

## 0. 当前项目状态

当前阶段：前端 MVP 打磨期 + Fastify 后端 MVP 稳定期。

已完成或基本完成：

- PC 端 Vue 前端雏形。
- 看板、物品资产、资产详情、记账、资金账户、我的/设置等主要页面。
- 本地 IndexedDB 数据存储。
- Fastify + TypeScript 后端骨架。
- SQLite 后端存储入口。
- 前端 HTTP 模式切换。
- 资产、附加项、记账、账户、分类的核心联动规则。
- 资产分类自定义、停用/恢复、筛选和排序。
- 后端 API 测试、前端服务测试和部分 E2E 测试。
- 后端入口已统一到 Fastify。

当前已知状态：

- 后端仍是单用户本地 API，未做登录和多用户同步。
- SQLite 当前仍以应用状态 JSON 保存，尚未拆成结构化表。
- 附件上传、正式认证、权限和云端部署暂未实现。

## 1. 阶段一：前端 MVP 收口

目标：让本地版成为可以长期试用、数据不容易乱的版本。

优先任务：

- 梳理所有表单必填校验和错误提示。
- 检查所有删除操作是否都有确认弹窗。
- 检查所有占用关系是否正确阻断删除。
- 优化资金账户页剩余细节。
- 优化记账日历视图和筛选体验。
- 清理临时 UI、调试日志和无用 mock 字段。

完成标准：

- 普通用户可以连续记录资产、记账、资金账户一周以上，不出现明显数据错乱。
- `npm.cmd test`、`npm.cmd run typecheck`、`npm.cmd run build` 通过。
- 涉及关键弹窗和交互时，补对应 E2E。

## 2. 阶段二：后端 MVP 稳定

目标：让 HTTP 模式和 IndexedDB 模式在核心业务规则上一致。

优先任务：

- 给 Fastify 路由补 schema 校验。
- 补齐后端错误码文档。
- 保持导入、导出、重置和 snapshot 稳定。
- 继续补账户、资产、记账、分类的边界测试。

完成标准：

- 前端可以稳定使用 `VITE_DATA_SOURCE=http`。
- 核心写入行为和 IndexedDB 模式一致。
- 后端测试覆盖主要资金联动规则。

## 3. 阶段三：SQLite 结构化

目标：把 SQLite 从单行 JSON 状态迁移成结构化表，降低后续查询、同步和多用户改造成本。

优先任务：

- 设计 `accounts`、`assets`、`asset_addons`、`transactions`、`account_flows`、`categories`、`settings` 表。
- 设计迁移脚本，从当前 `app_state` JSON 写入结构化表。
- 保持导入导出格式兼容。
- 给关键写入加事务测试。

完成标准：

- 结构化表能通过现有后端测试。
- 旧 SQLite 状态可迁移。
- API 响应不破坏前端。

## 4. 阶段四：认证和多设备

目标：从个人本地后端推进到真正全端可用。

优先任务：

- 用户注册、登录、当前用户 `/me`。
- Bearer token。
- 所有业务数据加 `userId` 隔离。
- 设计数据备份和恢复策略。

暂不做：

- 多用户协作。
- 复杂角色权限。
- 订阅支付。

## 5. 阶段五：上线准备

目标：从“能用”推进到“可放心长期使用”。

优先任务：

- 错误监控。
- 数据备份策略。
- 安全检查。
- 性能优化。
- 首次使用引导。
- 空状态和异常状态完善。
