# Rizhi 原型计划

## 1. 原型目标

本阶段目标是先完成 PC 网页端 MVP 的高保真静态原型，重点验证：

- 主导航是否清晰。
- 看板信息是否足够有用。
- 物品资产列表是否容易查找和筛选。
- 资产详情页是否能清楚表达“原始成本 + 附加项 + 日均成本”。
- 记账新增流程是否能自然关联资产和资金账户。
- 资金页是否能清楚区分资产账户和负债账户。

原型先以静态页面和基础交互为主，不要求完整数据写入。但页面结构、字段、按钮、弹窗、抽屉和状态要接近最终开发。

## 2. 原型范围

第一轮原型包含 5 个核心页面：

1. 看板页
2. 物品资产列表页
3. 物品资产详情页
4. 记账页
5. 资金页

第一轮原型包含 3 个关键操作面板：

1. 新增资产抽屉
2. 新增记账抽屉
3. 添加资产附加项抽屉

暂不做：

- 我的页完整设置。
- 登录注册。
- 云同步。
- 移动端页面。
- 复杂图表配置。
- 数据导入导出流程。

## 3. 全局布局

所有页面统一使用：

- 左侧固定侧边栏。
- 顶部工具栏。
- 主内容区。
- 右上角主操作按钮。

侧边栏导航：

- 看板
- 物品资产
- 记账
- 资金
- 我的

顶部栏：

- 全局搜索框：搜索资产、账户、记账。
- 通知图标。
- 用户头像和名称。

主内容区：

- 页面标题。
- 页面说明或关键统计。
- 筛选区。
- 内容卡片、表格或网格。

## 4. 看板页原型

### 4.1 页面目的

让用户打开应用后，快速知道当前财务和物品资产状态。

### 4.2 页面结构

第一行：数据概览卡片

- 净资产
- 总资产
- 总负债
- 本月收入
- 本月支出
- 预算剩余

第二行：提醒类卡片

- 即将过保的物品
- 即将还款的账户
- 闲置较久的物品

第三行：分析类卡片

- 本月收支趋势
- 日均成本最高的物品 TOP 5
- 最近记账记录

### 4.3 重点组件

- StatCard
- ReminderList
- MiniLineChart
- RankingList
- RecentLedgerList

### 4.4 示例数据

- 净资产：¥158,420.00
- 总资产：¥186,220.00
- 总负债：-¥27,800.00
- 本月收入：¥9,500.00
- 本月支出：¥3,620.00
- 预算剩余：¥1,380.00

## 5. 物品资产列表页原型

### 5.1 页面目的

让用户快速查看、搜索、筛选自己拥有或曾经拥有的物品资产。

### 5.2 页面结构

顶部：

- 页面标题：物品资产
- 主按钮：新增资产

分类 Tab：

- 全部
- 数码设备
- 衣物鞋子
- 家居用品
- 通勤器材
- 订阅服务

筛选区：

- 搜索框：搜索资产名称、品牌、型号
- 状态筛选：全部、使用中、闲置、已转卖、已报废
- 分类筛选
- 排序：购买日期、资产总成本、日均成本、过保日期
- 视图切换：卡片 / 表格

主体：

- 默认使用资产卡片网格。
- 每行 4 至 5 个卡片。

资产卡片字段：

- 资产图片
- 资产名称
- 品牌
- 分类
- 资产总成本
- 持有天数
- 日均成本
- 保修状态
- 当前状态
- 更多操作按钮

底部：

- 分页
- 每页数量选择

### 5.3 重点状态

- 使用中：绿色标签。
- 闲置：灰色标签。
- 即将过保：黄色标签。
- 已过保：红色标签。
- 订阅中：紫色标签。

## 9. 新增资产抽屉

### 9.1 触发入口

- 物品资产列表页右上角“新增资产”。
- 记账新增时选择“资产购买”后进入资产信息区。

### 9.2 表单字段

基础信息：

- 资产名称
- 分类
- 品牌
- 型号
- 图片

购买信息：

- 原始购买价
- 购买日期
- 付款账户
- 购买渠道
- 是否同步创建记账记录

保修信息：

- 保修开始日期
- 过保日期
- 预计使用年限

其他：

- 当前状态
- 当前估值
- 备注

### 9.3 操作按钮

- 取消
- 保存资产

保存后：

- 创建资产。
- 如果选择同步创建记账记录，则创建资产购买账单并影响资金账户。

## 10. 新增记账抽屉

### 10.1 触发入口

- 记账页右上角“新增记账”。
- 看板页快捷入口。
- 资产详情页添加附加项时可间接创建。

### 10.2 表单结构

第一段：交易基础信息

- 交易类型
- 金额
- 日期时间
- 分类
- 商家 / 备注

第二段：账户信息

- 付款账户
- 收款账户
- 转出账户
- 转入账户

不同交易类型显示不同账户字段：

- 支出：付款账户
- 收入：收款账户
- 转账：转出账户 + 转入账户
- 还款：还款账户 + 负债账户

第三段：资产关联

- 是否关联资产
- 选择已有资产
- 关联方式：
  - 仅关联为相关支出
  - 作为附加项并计入资产成本
  - 作为附加项但不计入资产成本
- 附加项名称
- 附加项类型

第四段：凭证与备注

- 图片或凭证
- 备注

### 10.3 关键交互

当交易类型选择“资产购买”：

- 显示资产信息区域。
- 保存时创建资产和账单。

当打开“关联资产”：

- 显示资产选择器。
- 用户选择资产后，可以选择是否创建附加项。

当选择“作为附加项并计入资产成本”：

- 显示附加项名称、类型、是否计入成本。
- 保存后更新资产总成本和日均成本。

## 11. 添加资产附加项抽屉

### 11.1 触发入口

- 资产详情页“添加附加项”。

### 11.2 表单字段

- 附加项名称
- 类型
- 金额
- 购买日期
- 付款账户
- 记账分类
- 是否计入资产成本
- 是否同步创建记账记录
- 备注
- 凭证图片

### 11.3 保存结果

保存后：

- 创建资产附加项。
- 创建对应记账记录。
- 资金账户扣款。
- 资产总成本重新计算。
- 日均成本重新计算。

## 13. 原型验收标准

原型完成后，应满足：

- 所有页面使用统一的侧边栏和顶部栏。
- 主色、按钮、卡片、表格、标签符合 [DESIGN.md](../design/DESIGN.md)。
- 用户能看懂资产总成本由哪些部分组成。
- 用户能看懂记账如何关联资产。
- 用户能看懂资金账户如何受记账影响。
- 关键按钮位置明确。
- 空状态和异常状态有基础表达。

## 14. 建议下一步开发动作

完成这份原型计划后，可以开始初始化 Vue 项目。

建议第一批开发：

1. 创建 Vue 3 + TypeScript + Vite 项目。
2. 接入 Vue Router 和 Pinia。
3. 落地 `tokens.css` 和 `global.css`。
4. 实现 `AppShell`、`Sidebar`、`Topbar`。
5. 用静态 demo 数据搭出看板页。
6. 再做资产列表和资产详情页。
